# SCGP 情绪行为调节模块 PRD

> **文档类型**: 模块级 PRD (含技术约束)
> **产品名称**: `SCGP / 星愿能力发展平台`
> **所属模块**: `emotional` (情绪行为调节)
> **编写时间**: 2026-03-16
> **系统基线**: Electron + Vue 3 + TypeScript + SQL.js (本地优先)

## 1. 文档目标

本文档用于在 SCGP 现有系统内新增“情绪行为调节模块”，明确其产品定位、功能范围、页面流程、数据设计、交互要求与验收标准，指导后续基于 AI 辅助（如 Claude）的设计、开发、测试与验收。

## 2. 项目背景

SCGP 当前已具备学生管理、能力评估、游戏训练、训练记录、报告管理和本地资源管理能力，是一个成熟的本地优先特殊教育平台。
目前系统在模块体系中已预留 `emotional` 模块，但仍处于占位状态。随着教学需求扩展，系统需要增加面向特殊儿童（ASD、ID 等）的情绪识别、社交推理和表达关心训练能力，形成从“认知-推理-回应-数据沉淀”的完整闭环。

## 3. 产品定位

情绪行为调节模块是 SCGP 中的一个正式训练模块，服务于特殊教育场景中的情绪理解与社交回应干预。
*注：首期版本侧重于“情绪认知与社交技能训练”，为儿童提供“情绪识别”和“社交回应”的结构化练习框架。*

模块核心价值包括：
- 帮助儿童识别常见基础情绪（结合情绪色彩分区 Zones of Regulation）。
- 帮助儿童理解情绪产生的场景原因与核心需求。
- 帮助儿童学习表达关心的多种方式及换位思考（心智理论 Theory of Mind）。
- 帮助教师和家长追踪学生训练表现与进步趋势。

## 4. 用户与使用场景

### 4.1 目标用户
- 特殊学校学生（孤独症谱系、发育迟缓、社交理解困难儿童）
- 教师（特教老师、康复师）
- 家长

### 4.2 核心使用场景
- **课堂/个训教学**：教师通过大屏或桌面端引导学生进行场景化推演。
- **家庭康复陪练**：家长在家庭自然情境中，利用系统复盘类似事件。
- **数据追踪复盘**：教师/家长查看孩子在不同情绪下的理解偏差与话术偏好。

## 5. 产品目标

1. 建立生活化、场景化的情绪识别训练能力。
2. 建立从“看见情绪”到“理解原因”再到“做出回应”的阶梯式训练路径。
3. 让学生学习表达关心的多种方式及其对他人产生的影响。
4. 建立情绪行为训练的本地化数据记录和多维度趋势分析能力。
5. 保证训练过程符合特殊教育“阶梯式干预（Least-to-Most Prompting）”和“低认知负荷”原则。

## 6. 模块范围

### 6.1 首期范围 (MVP)
首期版本包含两个核心训练子模块：
1. **情绪与场景** (Emotion & Scene)
2. **表达关心** (Expressing Care)

### 6.2 非首期范围
- 即时情绪安抚工具箱（如深呼吸/平静角，拟入二期）
- AI 自由对话生成与联网素材库
- 复杂语音识别评分与云端同步

## 7. 功能需求

### 7.1 子模块一：情绪与场景

#### 7.1.1 场景识别
系统需支持不少于 80 个真实生活场景资源位。首期内置少量演示数据，但底层必须支持按类型扩展。
- **场景覆盖**：生日派对、医院探望、学校课堂、同伴争抢、摔倒受伤、被拒绝等。
- **功能要求**：支持场景大图展示；支持点击切换；支持按难度和目标分类查询。

#### 7.1.2 情绪解读 (引入情绪色彩分区)
系统引导学生在 5 种基础情绪中进行选择，**必须结合视觉色彩双重编码**以降低认知负荷：
- 🟢 **开心** (绿色区 - 能量适中，适合学习)
- 🔵 **失落** (蓝色区 - 能量低)
- 🟡 **尴尬** (黄色区 - 能量升高，需注意)
- 🔴 **生气** (红色区 - 能量过高)
- 🔴 **害怕** (红色区 - 能量过高)

**功能要求**：
1. 选择后立即反馈（正向强化或温和纠正）。
2. 错误时触发阶梯式提示（第一步：视觉线索闪烁；第二步：排除错误干扰项；第三步：直接示范）。

#### 7.1.3 社交推理
情绪识别正确后，展示 2-3 个与场景强绑定的引导问题。
- **问题维度**：原因推理（他怎么了？）、需求推理（他需要什么？）、自我代入（如果是你？）。
- **功能要求**：每题需有标准答案/可接受答案；支持温和的错误反馈说明。

#### 7.1.4 问题解决
提供 3-4 种场景应对方案供选择。
- **功能要求**：每个方案需有明确的适宜性判定（最佳/可接受/不合适）；**选项必须采用“图文双重编码”（Icon+文字）**；选择后展示对他人影响的解释说明。

### 7.2 子模块二：表达关心

#### 7.2.1 情境练习
系统需支持不少于 60 个“需要表达关心”的场景资源位。
- **典型场景**：朋友考试失利、同伴哭泣、家长疲惫等。

#### 7.2.2 话术训练
每个场景提供三类关心表达方式：
- **共情式**（例：“我知道你现在很难过”）
- **建议式**（例：“我们要不要去喝杯水？”）
- **行动式**（例：递上一张纸巾）
- **功能要求**：选择后展示“使用效果说明”（解释该表达给对方带来的感受）；同一场景下不同表达方式可比较。

#### 7.2.3 视角切换 (心智理论 ToM 训练)
支持两种交互视角：
1. **表达者视角**：用户选择自己会说的话 -> 系统展示接收者虚拟反应（文字+简单表情）。
2. **接收者视角**：系统展示不同话术 -> 用户选择“听起来最舒服”的表达 -> 系统说明原因。

#### 7.2.4 进度追踪
自动记录：场景ID、子模块类型、正误、提示次数、重试次数、答题时间、偏好话术类型、视角切换结果。

## 8. 报告需求

新增情绪模块专属可视化报告页，图表组件统一使用 ECharts，数据来源于本地 SQLite 记录。
1. **情绪识别正确率趋势图**（折线图）
2. **各类场景掌握情况分布图**（雷达图）
3. **不同情绪类型识别表现图**（柱状图，需带情绪主色调）
4. **关心话术类型偏好图**（饼图，共情/建议/行动占比）
5. **表达者与接收者视角表现对比图**（双柱状图）
6. 老师/家长可读的干预建议（基于错误模式自动生成简短文本）。

## 9. 特殊教育交互设计要求 (UI/UX)

1. **单屏单任务**：避免并列复杂信息，当前焦点高亮，背景适度压暗。
2. **视觉双重编码**：所有选项必须“左图右文”或“上图下文”，不能仅有纯文字。
3. **情绪色彩规范**：严格遵守情绪对应的颜色标准（绿/蓝/黄/红），按钮和反馈边框需与情绪颜色呼应。
4. **防误触与触屏友好**：按钮大，点击区域清晰，选项间距拉大。
5. **无惩罚性反馈**：错误时不使用刺耳音效或红色大叉，使用“温和提示音”及“再试试看”的中性文案。
6. **阶梯式提示机制**：
   - 第一次错误：轻提示（错误项稍微变灰）。
   - 第二次错误：缩小范围（隐藏一半错误项）或高亮视觉线索。
   - 第三次错误：展示正确答案并提供辅导语音/文本。
7. **低感官刺激**：避免高刺激动态效果和强闪烁。

## 10. 数据设计约束 (面向 AI 架构说明)

为了与 SCGP 底座良好结合，本模块的数据模型需使用 TypeScript 接口严格定义，并存储于现有的 `sys_training_resource` 和训练记录表中。

### 10.1 资源定义 (Resource Schema)
资源表 `sys_training_resource` 的 `ext_data` 字段将存储具体的训练 JSON：

```typescript
// 1. 情绪与场景训练结构 (emotion_scene)
interface EmotionSceneResource {
  scene_id: string;
  title: string;
  image_url: string; 
  difficulty_level: 1 | 2 | 3;
  target_emotion: 'happy' | 'sad' | 'embarrassed' | 'angry' | 'scared';
  // 第一阶段：情绪解读
  clues: string[]; // 视觉线索，如 ["紧锁的眉头", "握紧的拳头"]
  // 第二阶段：社交推理
  reasoning_questions: {
    question_id: string;
    question_type: 'cause' | 'need' | 'empathy'; // 原因/需求/换位思考
    question_text: string;
    options: {
      id: string;
      text: string;
      image_url?: string;
      is_correct: boolean;
      feedback_text: string; // 温和反馈说明
    }[];
  }[];
  // 第三阶段：问题解决
  solutions: {
    id: string;
    text: string;
    image_url: string; // 必须有图
    suitability: 'optimal' | 'acceptable' | 'inappropriate'; // 适宜度判定
    explanation: string; // 选择后的效果说明
  }[];
}

// 2. 表达关心训练结构 (care_scene)
interface CareSceneResource {
  scene_id: string;
  title: string;
  image_url: string;
  care_type: 'empathy' | 'advice' | 'action'; // 共情/建议/行动
  // 视角一：表达者
  sender_options: {
    id: string;
    text: string;
    receiver_reaction_text: string;
    receiver_reaction_emoji: string;
  }[];
  // 视角二：接收者
  receiver_options: {
    id: string;
    text: string;
    is_comforting: boolean;
    reason_text: string;
  }[];
}
```

复用现有的训练记录主表，将其过程数据存储于 `raw_data` 字段：

    is_correct: boolean;

    retry_count: number;
    response_time_ms: number;
  }
