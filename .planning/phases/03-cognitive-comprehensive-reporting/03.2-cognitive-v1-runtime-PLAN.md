# Plan 03.2: Cognitive V1 Runtime Integration

## Goal

Wire the new `cognitive` scale through the shipped assessment flow so a user can launch it, complete it, save it, and reopen its report from the report center.

## Context

- The shared runtime already handles question flow, scoring, feedback, and completion.
- The repo still requires explicit registration for driver lookup, entry points, persistence branching, report routing, and report-center display.
- This plan should stay within the generic container path unless the foundation proves a special pre-form is unavoidable.

## Requirements Covered

- MOD-03: User can conduct cognitive assessments using the ScaleDriver architecture.
