## Gaps Identified

### Gap 1: Missing "Part 1: General Information" Section
**Location:** SocialForm.vue
**Issue:** The form starts directly with Part 2 (Social Competence), skipping Part 1 (General Information)
**Required Fields:**
- 填表者 (Reporter): 父 / 母 / 其他人
- 其他人关系 (Other relationship text field, shown when "其他人" selected)
- 父亲职业 (Father occupation) with hint text from MD doc
- 母亲职业 (Mother occupation) with hint text from MD doc

### Gap 2: Part 2 Qualitative Design Not Implemented (CRITICAL)
**Location:** SocialForm.vue - All sections I-IV
**Issue:** Current implementation asks users to select "count" (how many hobbies/organizations), but the original CBCL design requires:
- **Section I (Sports):** 3 text fields (a, b, c) for specific sport names + "No hobby" checkbox that disables text fields
- **Section II (Hobbies):** Same pattern - 3 text fields + "No hobby" option
- **Section III (Organizations):** 3 text fields + "Not participated" option
- **Section IV (Labor):** 3 text fields + "None" option
- **Dynamic calculation:** System should auto-calculate count from filled text fields, NOT ask user to select count

**Specific Options from MD:**
- 交友数量: "无 / 1个 / 2—3个 / 4个及以上"
- 见面频率: "不到一次 / 1—2次 / 3次及以上"
- 时间多少: "不知道 / 较少 / 一般 / 较多"
- 活跃程度: "不知道 / 较差 / 一般 / 较高"
- 做事质量: "不知道 / 较差 / 一般 / 较好"

### Gap 3: Conditional Fields for School Performance Missing
**Location:** SocialForm.vue - Section VII
**Issue:** Missing conditional text fields:
- When "在特殊教育班级" checked → show "性质" text field
- When "留过级" checked → show "几年级" and "理由" text fields
- When "有学习或其他问题" checked → show:
  - "问题内容" textarea
  - "何时开始" text field
  - "是否解决" radio + "何时解决" text field (if solved)

### Gap 4: Data Structure Mismatch
**Location:** src/types/cbcl.ts - CBCLSocialCompetenceData
**Issue:** Current structure expects count values, but should support:
- `sports_list: string[]` (3 items)
- `hobbies_list: string[]` (3 items)
- `organizations_list: string[]` (3 items)
- `labor_list: string[]` (3 items)
- `father_occupation: string`
- `mother_occupation: string`
- `reporter: 'father' | 'mother' | 'other'`
- `other_relation: string`

### Gap 5: Question 113 Multi-Text Support
**Location:** cbcl-questions.ts, QuestionCard.vue
**Issue:** Question 113 requires 3 separate text inputs (问题1, 问题2, 问题3), but current hasDescription only supports single text field

<decisions>
## Implementation Decisions

### UI Design
- Use Element Plus Form with reactive validation
- Implement "None/No" checkbox that conditionally disables/enables text fields
- Use computed properties to auto-calculate counts from text field arrays
- Conditional rendering for school performance follow-up questions

### Data Flow
1. User fills qualitative text fields (e.g., "游泳", "跑步")
2. System auto-calculates count (e.g., 2 items filled → count = 2)
3. Count passed to scoring engine (existing logic unchanged)
4. Raw text data preserved for reference in database

### Bridge Logic Location
- Add transformation logic in CBCLDriver or before submission
- Keep scoring engine unchanged (it still receives count values)
- Add helper method: `calculateCountFromTexts(texts: string[]): number`

### Question 113 Solution
- Split into 113a, 113b, 113c in cbcl-questions.ts
- Or enhance QuestionCard to support multiple description fields
- Store as array in behavior_raw_answers

</decisions>

### Auto-Calculation Logic
```typescript
function calculateCount(texts: { a: string, b: string, c: string, none: boolean }): number {
  if (none) return 0
  return [texts.a, texts.b, texts.c].filter(t => t.trim().length > 0).length
}
```
