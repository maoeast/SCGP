## Technical Decisions

### Why Split Question 113?
The original CBCL source document shows Question 113 with 3 separate text inputs (问题1, 问题2, 问题3). Splitting into sub-items (113a, 113b, 113c) allows each to have its own score (0-2) and description, matching the clinical design.

### Bridge Logic Location
Placed in CBCLDriver rather than form component to:
1. Keep form component pure UI
2. Allow reuse if data comes from other sources
3. Maintain single source of truth for scoring logic

### Backward Compatibility Strategy
JSON column storage in SQLite allows flexible schema evolution. The migration helper ensures old records load without errors, displaying empty text fields where original data didn't capture qualitative inputs.

## Risks and Mitigations

| Risk | Mitigation |
|------|------------|
| Existing assessments break | Migration helper auto-converts old format |
| Scoring engine incompatible | Bridge logic preserves count fields |
| Form validation too strict | Conditional validation only triggers when fields are visible |
| Data loss during migration | Old fields preserved, new fields get safe defaults |
