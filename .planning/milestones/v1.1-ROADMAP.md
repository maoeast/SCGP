# Milestone v1.1: Emotional Authoring & Scene Gallery

**Status:** ✅ SHIPPED 2026-03-17  
**Phases:** 06-08  
**Total Plans:** 3

## Overview

v1.1 focused on emotional-module usability. The milestone replaced raw JSON authoring in Resource Center with dedicated visual editors, then added selector galleries so teachers can choose a specific emotional scene before training begins.

## Phases

### Phase 6: Emotional Resource Contract & Editor Infrastructure

**Goal**: Lock the current emotional metadata contract and prepare reusable parsing, normalization, and validation utilities for visual editing.  
**Depends on**: —  
**Plans**: 1 plan

Plans:

- [x] 06-01: Emotional Resource Contract & Editor Infrastructure

**Details:**
- Existing `emotion_scene` and `care_scene` resources can be loaded into typed editor state without exposing raw JSON to users.
- Save-time normalization outputs metadata compatible with `EmotionSceneTraining.vue` and `CareExpressionTraining.vue`.
- Validation covers nested prompts, options, solutions, utterances, and receiver options with teacher-readable error messaging.
- No database schema migration is required to keep v1.0 demo/custom emotional resources usable.

### Phase 7: Visual Emotional Resource Editors

**Goal**: Replace emotional resource JSON textareas in Resource Center with dedicated visual editors.  
**Depends on**: Phase 6  
**Plans**: 1 plan

Plans:

- [x] 07-01: Visual Emotional Resource Editors

**Details:**
- Admin can create and edit `emotion_scene` resources through a dedicated form component.
- Admin can create and edit `care_scene` resources through a dedicated form component.
- Emotional resource authoring no longer exposes raw `meta_data` textareas in normal create/edit flows.
- Editor UI supports dynamic add/remove flows for nested options and feedback fields using Element Plus form controls.

### Phase 8: Emotional Scene Gallery & Launch Flow

**Goal**: Add selector pages so teachers choose a concrete scene before entering emotional training runtime.  
**Depends on**: Phase 7  
**Plans**: 1 plan

Plans:

- [x] 08-01: Emotional Scene Gallery & Launch Flow

**Details:**
- `情绪与场景` and `表达关心` menu cards route to dedicated selector pages instead of entering runtime directly.
- Selector pages list all active scenes in a card grid rather than auto-loading the first available record.
- Scene cards display cover, title, difficulty, and emotion color cues in a touch-friendly layout that matches PRD section 9.
- Clicking a card navigates into the existing training runtime with explicit `resourceId` while preserving student context.
- Empty states clearly guide teachers back to Resource Center when no valid scenes are available.

## Milestone Summary

**Key Decisions:**

- Preserve `sys_training_resource.meta_data` as the persistence field instead of reopening schema scope.
- Use the current `src/types/emotional.ts` contract as the source of truth for editor serialization.
- Keep selector flow inside the current static router rather than reopening dynamic routing/platformization work.

**Issues Resolved:**

- Emotional resource authoring no longer requires raw JSON editing in Resource Center.
- Emotional runtime launch no longer assumes “take the first row” for normal teacher flow.
- Teachers can visually choose a specific scene before entering training.

**Issues Deferred:**

- Batch import/export of preset emotional resources.
- Emotional report polish based on richer scene taxonomy.
- Broader route/menu platformization beyond the current static emotional module scope.

**Technical Debt Incurred:**

- Repository-wide historical TypeScript errors remain outside the v1.1 touched-file scope.
