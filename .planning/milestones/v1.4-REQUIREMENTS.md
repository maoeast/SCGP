# REQUIREMENTS

## Milestone v1.4: Dashboard Special Ed Command Center

**Goal:** Rebuild the homepage into a special-education daily command center driven only by real local data, with direct launch from today's schedule into actual training runtime.

## Completed Requirements

### Real Data Dashboard

- [x] **DASH-01**: The homepage top cards use only real local data for student count and pending assessment reminders.
- [x] **DASH-02**: Today's task count is derived from active `sys_training_plan` rows whose date range includes today.
- [x] **DASH-03**: Weekly anomaly count is derived from real training data, using low `accuracy_rate` and high emotional-session `hint_level` evidence from the last 7 days.

### Business Panels

- [x] **DASH-04**: Today's Schedule replaces "recent students" and lists only real active plans joined with real student records.
- [x] **DASH-05**: The anomaly panel uses real `training_records` plus `emotional_training_session` / `emotional_training_detail` aggregation; if no anomalies exist, the UI shows a stable-state empty message instead of mock alerts.
- [x] **DASH-06**: Smart Alerts identifies students with no assessment history or no assessment in the last 6 months by aggregating the existing assessment tables.

### Direct Launch Flow

- [x] **DASH-07**: Clicking `开始训练` from Today's Schedule launches the real target training page directly with `studentId`, `planId`, and resource context, without forcing the teacher back through student/module selection.
- [x] **DASH-08**: Training pages reached from Dashboard or Plan preserve source context and return to the correct upstream entry (`/dashboard` or `/training-plan`) on exit.
