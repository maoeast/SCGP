# REQUIREMENTS

## Milestone v1.3: Unified Assessment Word Export

**Goal:** Let teachers export the main assessment reports as clean editable `.docx` files through one consistent export flow.

## Completed Requirements

### Export Foundation

- [x] **WORD-01**: System provides a generic `docx`-based Word export renderer that accepts normalized report payloads instead of handwritten per-page `docx` trees.
- [x] **WORD-02**: The new exporter uses the existing `docx` and `file-saver` stack only; no new runtime dependency is introduced.

### Scale Migration

- [x] **WORD-03**: SDQ and CBCL report pages export `.docx` through the universal payload-based exporter.
- [x] **WORD-04**: SRS2 and CSIRS report pages export `.docx` through the universal payload-based exporter.
- [x] **WORD-05**: Conners PSQ and Conners TRS report pages export `.docx` through the universal payload-based exporter.
- [x] **WORD-06**: S-M and WeeFIM report pages migrate from their old dedicated Word pipeline to the universal exporter without losing core report content.

### UX Cleanup

- [x] **WORD-07**: Legacy "导出PDF" / "打印报告" export entry points are removed from the migrated assessment report pages so the UI consistently exposes "导出Word".

- **WORD-08**: Generic assessment Word payloads support embedded chart images and page-break tuning.

## Notes

- v1.3 deliberately standardized on Word-first export rather than revisiting PDF quality.
- v1.3 did not add any new export dependency beyond the existing `docx` and `file-saver` stack.
